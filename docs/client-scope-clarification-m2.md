## 1. 決済エラーについて

「Edge Function returned a non-2xx status code」というエラーの原因を特定いたしました。

運営グループのStripe決済連携（Stripe Connect）が、銀行口座・本人確認情報の登録まで完了していないことが直接の原因です。この登録フォームは元々 **Milestone 4** で実装予定の機能でしたが、決済確認を先に進めていただくため、テストモードで1グループ分の連携を完了させる対応を進めています。

## 2. 「1グループを完成形に」というご要望について

お送りいただいたプロトタイプを拝見し、現在の実装との差分を確認いたしました。
結論から申し上げますと、**プロトタイプに含まれる一部の機能は、Milestone 1で確定した技術仕様書（M1_技術仕様書）の範囲外**であることが分かりました。具体的には以下の機能です。

| 機能 | プロトタイプ | M1〜M4の合意範囲 |
|---|---|---|
| グループ切り替え（複数グループの管理） | ○ | × |
| メンバー選択（購入時にメンバーを指定） | ○ | × |
| 複数アイテムのカート機能 | ○ | × |
| ポイント制度（獲得・交換・申請管理） | ○ | × |

これらは見た目の調整ではなく、**データベース設計の追加（グループ・メンバー・ポイントのテーブル新設）、決済ロジックの拡張、関連する複数画面の新規実装**が必要な機能群です。現在のM2の範囲（認証・決済・チケット発行）には含まれておらず、追加の開発範囲・期間として扱う必要があると判断いたしました。
