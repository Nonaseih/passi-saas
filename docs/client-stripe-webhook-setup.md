PassI の決済機能実装フェーズにあたり、Stripe アカウントからもう一点情報をご提供いただく必要があります。**Webhook 署名シークレット**と呼ばれるキーです。

Stripe ダッシュボード上での作業で、所要時間は約 3 分です。

---

## ご提供いただきたいもの

Stripe が発行する Webhook 署名シークレット（`whsec_` から始まる文字列）です。  
これにより、サーバーが受け取る決済完了通知が本当に Stripe から送られたものかどうかを検証できます。

---

## 取得手順

1. **dashboard.stripe.com** にログインしてください
2. 左サイドバーから **「開発者」→「Webhook」** を開きます
3. **「エンドポイントを追加」** をクリックします
4. **「エンドポイント URL」** 欄に以下を入力してください：
   ```
   https://isevhcauafjdxspyqfvu.supabase.co/functions/v1/stripe-webhook
   ```
5. **「リッスンするイベントを選択」** で以下の 3 つを追加します：
   - `checkout.session.completed`
   - `payment_intent.payment_failed`
   - `account.updated`
6. **「エンドポイントを追加」** をクリックして保存します
7. 次の画面で **「署名シークレット」** の横にある **「表示」** をクリックします
8. 表示された `whsec_` から始まる値をコピーしてください

コピーした値をお送りいただければ、残りの設定はこちらで対応いたします。

---

## テストモードについて

Stripe ダッシュボード左上のトグルが **「テストモード」** になっている場合は、そのままテストモードでエンドポイントを追加してください。本番リリース時に改めてライブモードで同じ手順を行います。
